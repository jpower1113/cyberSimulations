About our Site

This dashboard is a security testing and monitoring interface designed to demonstrate how common web attack patterns are detected and logged in a controlled environment. It is intended for use only against the team’s own virtual machine (VM) endpoints for educational, defensive, and validation purposes.

By generating simulated traffic that resembles real-world attack techniques, the dashboard provides immediate visibility into how application-layer requests propagate through the web stack and how network-based detection tools respond.

End-to-End Detection Workflow

The dashboard demonstrates a complete security monitoring pipeline:

  1. Run a traffic simulation from the dashboard
  2. Traffic reaches the Apache/Flask web application
  3. Suricata inspects the traffic and generates alerts
  4. Apache access logs record the requests
  5. The dashboard surfaces recent alerts and logs for review

This workflow provides clear evidence that simulated traffic is successfully detected and correlated across multiple layers of the system.

Simulation Sections

SQL Injection–Like Input Patterns

This section sends input patterns that resemble common SQL injection attempts. These requests are used to validate that database-oriented attack signatures are properly detected by Suricata and logged by the web server.

XSS–Like Payload Patterns

This section generates requests containing patterns similar to cross-site scripting (XSS) payloads. The goal is to observe how script-like input is handled by the application and flagged by intrusion detection rules.

Directory Traversal Attempts

This section simulates path traversal requests that attempt to access files outside of the intended web directory structure. These requests help verify detection of unauthorized file access attempts.

HTTP Load Spike

This section creates a short burst of HTTP requests to simulate a small load spike. It is useful for observing how elevated request rates appear in access logs and whether traffic-based alerts are triggered.

Viewing Results

After running any simulation, the dashboard displays recent Suricata alerts and Apache access logs. These outputs provide immediate feedback and allow users to correlate simulated activity with detection events.

Together, these views demonstrate how defensive monitoring tools can identify, log, and surface suspicious behavior in real time.

Note: This dashboard is intended strictly for authorized testing within controlled environments. All simulations should be run only against systems owned or explicitly permitted by the team.
